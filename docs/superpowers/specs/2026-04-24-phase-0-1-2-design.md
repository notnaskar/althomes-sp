# AltHomes — Phase 0, 1, 2 Execution Design

**Date:** 2026-04-24  
**Scope:** Phase 0 (Fork & Project Setup) → Phase 1 (Sanity Schema) → Phase 2 (Route Architecture & Data Layer)  
**Approach:** Strict sequential — each phase fully committed before the next begins  
**Package manager:** npm (bun.lock deleted, package-lock.json committed)

---

## Constraints (Non-Negotiable)

- `output: 'standalone'` added before any other code change — required for Coolify/Hetzner
- `<SanityLive />` in `(site)/layout.tsx` only — never in root layout
- All GROQ queries wrapped in `defineQuery()` — plain groq strings break TypeGen silently
- `browserToken` in `defineLive` must be Viewer-scoped only — confirmed via `SANITY_API_READ_TOKEN`
- Route paths from Figma/master doc: `/our-homes/[slug]`, not `/properties/[slug]`
- No Vercel-specific APIs (`@vercel/kv`, `@vercel/blob`, Edge Runtime) — production target is Hetzner + Coolify
- `legalPage` is multi-instance — intentionally excluded from singleton duplication prevention

#### 6. Prevent Singleton Duplication — `sanity.config.ts`

```ts
document: {
  newDocumentOptions: (prev, { creationContext }) => {
    const singletons = [
      'homePage', 'ourHomesPage', 'altWayPage', 'experiencesPage',
      'joinUsPage', 'contactPage', 'site'
    ]
    if (creationContext.type === 'global') {
      return prev.filter(t => !singletons.includes(t.templateId))
    }
    return prev
  },
}
```

`legalPage` intentionally NOT in this list.

## Phase 2 — Route Architecture & Data Layer

**Goal:** All routes defined. Data fetching wired to Sanity. Build passes clean with TypeGen types.

**Dependency:** Phase 1 TypeGen output committed + test data seeded

### Tasks (in order)

#### 1. Route Structure

Replace existing `[[...slug]]` catch-all with explicit routes under `src/app/(site)/`:

```
(site)/
├── layout.tsx              ← verify SanityLive present
├── page.tsx                ← homepage (replace catch-all)
├── our-homes/
│   ├── page.tsx
│   └── [slug]/page.tsx
├── the-alt-way/page.tsx
├── experiences/page.tsx
├── join-us/page.tsx
├── contact/page.tsx
├── [slug]/page.tsx         ← legal pages — notFound() if slug not in legalPage docs
├── blog/
│   ├── page.tsx
│   └── [slug]/page.tsx    ← keep existing module renderer
```

`not-found.tsx` at `src/app/` root (not inside `(site)/`).

#### 2. GROQ Queries — `src/sanity/lib/queries.ts`

All wrapped in `defineQuery()`:

```ts
siteSettingsQuery
homePageQuery
ourHomesPageQuery
allPropertiesQuery          // status == "active", order by displayOrder
propertyBySlugQuery         // dereferences amenities[]->, experiences[]->, inline reviews[] filtered published==true ordered stayDate desc
allExperiencesQuery         // dereferences properties[]->
altWayPageQuery
featuredReviewsQuery        // featured == true && published == true
experiencesPageQuery
joinUsPageQuery
contactPageQuery
legalPageBySlugQuery
allPostsQuery
postBySlugQuery             // includes modules
```

#### 3. Data Fetching Functions — `src/sanity/lib/data.ts`

One function per query using `sanityFetch` from `live.ts`. Returns typed data from TypeGen output.

#### 4. `generateMetadata` on Every Page

Pattern:
```ts
export async function generateMetadata({ params }): Promise<Metadata> {
  // fetch page data
  return {
    title: data?.seo?.metaTitle ?? data?.title ?? 'AltHomes',
    description: data?.seo?.metaDescription,
    openGraph: { images: data?.seo?.ogImage ? [urlFor(data.seo.ogImage).url()] : [] },
  }
}
```

No page has a missing or default title.

#### 5. Legal Page Guard

`(site)/[slug]/page.tsx` must call `notFound()` when `legalPageBySlugQuery` returns null — prevents unknown slugs from hitting this route silently.

#### 6. Verify Build

```bash
npm run build
```

TypeGen runs via `prebuild`. Zero type errors required.

#### 7. Commit

`feat: route architecture and data layer`
